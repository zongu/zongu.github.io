RabbitMQ-Queue-TTL

  • Carter Ho
  • 3 Minutes
  • 2019年6月22日
  • RMQ的Queue跟Kafka的Partition都會保存訊息直到Consumer來取走，來確保資料送達，但如果在大量訊息壓力下轟炸任誰也承受不住，所以必須要對Queue裡的訊息做保留策略。
  • 一般的情況下我們會保留訊息一天，如果線上出了問題反映後還能有餘裕時間把queue裡的資料消化掉，所以機器配置至少要能承受一天的訊息量；當然是針對高頻率的訊息佇列。
  • 設置TTL方式
    • rabbitctl
    • http
    • Per-Message TTL
    • Queue TTL
    • 以上設定最後會以最小TTL為準
